Source Offline
Sources ›
Turn each playback source (AirPlay, Bluetooth, Spotify Connect) on or off.
Speaker name ›
Change the name shown in AirPlay, Spotify Connect, Bluetooth, and USB Audio.
Voice provider ›
Choose which real-time voice backend the speaker uses (Gemini, OpenAI, or Grok) and paste API keys.
Wake word ›
Pick which phrase wakes the speaker — “Jarvis”, “Hey Jarvis”, “Alexa”, or “Hey Mycroft”. New models can be added by updating jasper/wake_models.py.
AirPlay sync mode ›
Synced (default — works for music, video A/V, and multi-room) or free-running (fallback for DAC-specific issues).
Integrations ›
Connect external services to your speaker — Spotify accounts, Google Calendar + Gmail, and Home Assistant for smart-home control.
Accessories ›
Onboard a wireless accessory — currently the ESP32 rotary dial; the AMOLED touch satellite is in progress.
Bluetooth ›
Pair phones (to play music to the speaker), volume knobs, headphones, anything Bluetooth. Toggle the adapter on/off and whether the speaker is discoverable.
Wi-Fi ›
Switch which Wi-Fi network the speaker is on, see signal strength of nearby networks, and forget saved ones.
Transit ›
Configure nearby subway and bus stops so you can ask “when's the next train?” or “next bus?”. Enter your address; the wizard finds the closest stops via OpenStreetMap. NYC subway is keyless; NYC bus needs a free MTA BusTime API key.
Weather ›
Set the default location and units for weather questions when you do not name a city.
Speaker peering ›
Off by default. When you have multiple JTS speakers on the same network, turn this on so only one responds to each wake word instead of all of them at once.
Room correction ›
Measure your room from your iPhone and apply correction filters to CamillaDSP. Browser will warn "Not Private" the first time — see the note below.
CamillaDSP ›
Tune the speaker's audio processing — filters, mixers, signal levels, and live config switching. Opens in a new tab.
System ›
Live view of what the speaker is doing — memory, CPU, temperature, cloud activity (sessions, tokens, est. cost), and quick actions (restart voice, run diagnostics, reboot).
Wake-word corpus recorder ›
Tooling / debug page. Browser-driven recording of "Jarvis" utterances across distance + music conditions for the wake-word retraining experiment. Stops jasper-voice for the duration of the session; click-to-record, instant playback, one-click delete. See docs/HANDOFF-wake-training-experiment.md Phase 0b.

About the "Not Private" warning on Room correction: capturing microphone audio in a browser requires HTTPS. This speaker uses a self-signed certificate (no public CA — it's on your home network), so Safari and Chrome warn the first time.

Either tap Show Details → visit this website to dismiss the warning (works fine; reappears every visit), or download the JTS root certificate and trust it once on this device to silence the warning permanently. The room-correction page itself walks you through the trust steps.
