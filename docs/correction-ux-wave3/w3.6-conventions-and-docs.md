## Context

Punchlist item #41 (`campaign/punchlist.md`) is a hardware-found bug,
already fixed (#1537): the shared stylesheet's only hiding mechanism
is the native `[hidden]` attribute selector —

```css
[hidden] { display: none !important; }
```

(`deploy/assets/app.css:137`) — there is no `.hidden` CLASS rule in
`app.css`. Before the fix, the crossover page hid stale controls with
`class="... hidden ..."` / `classList.add('hidden')`, which rendered
fully visible because nothing in the shared stylesheet targets that
class name. `tests/test_web_wizard_conventions.py` now enforces the
native-attribute convention going forward
(`test_static_modules_hide_elements_with_the_attribute_not_a_class`,
`test_wizard_markup_hides_elements_with_the_attribute_not_a_class`),
with one legacy exemption:

```python
_LEGACY_HIDDEN_CLASS_PAGES = {
    "jasper/web/correction_setup.py": "deploy/assets/correction/correction.css",
    "deploy/assets/correction/js/main.js": "deploy/assets/correction/correction.css",
}
```

This exemption is **file-granular**: any render path inside
`jasper/web/correction_setup.py` — a single file that serves the room
flow, the bonded-follower delegation page, and (per the module
docstring) balance/sync routes too — is exempt from the
attribute-not-class check, even though only the specific room-page
markup that legitimately owns a local `.hidden` rule in
`correction.css` needs the exemption. The room follower page
(`_render_follower_page`) is a concrete example the review already
named: it rides the same file-wide exemption without anyone having
confirmed it actually needs `.hidden`-class hiding rather than the
native attribute every other page in the codebase now uses.
